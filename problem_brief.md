# 一汽物流“备车 + 短驳”项目背景摘要

## 1. 项目来源

本项目来源于“长春国际汽车城&一汽物流杯”第八届全国大学生物流设计大赛案例文件。原案例文件提供了多个可选题目，参赛队伍需要任选 2 个以上题目构成参赛主题。

本项目节选并融合了其中三个方向形成综合方案，其中：

- 案例 14：一汽物流智慧物流园整车库内管理与优化设计；
- 案例 16：如何实现整车售前短驳运输“零公里”；
- 第三个方向用于补充绿色物流、数智化或绩效评价等综合分析视角。

需要特别说明：

- 本项目在案例 14 中重点做的是“备车问题”，不是完整覆盖案例 14 的所有库区管理问题；
- 本项目在案例 16 中重点做的是“短驳问题”，围绕整车售前短驳运输“零公里”目标进行方案选型、成本效益分析和绩效评价；
- 当前 Web 项目是从比赛方案中提炼出的简历展示型系统，不等同于真实企业生产系统；
- 项目中使用的数据均为模拟数据或脱敏示例数据，不使用真实企业内部数据。

---

## 2. 项目名称

一汽绿运：备车 + 短驳物流调度成本评估平台

---

## 3. 项目定位

本项目是一个面向整车物流场景的 B/S 架构 Web Demo，目标是将比赛方案中的业务分析、运筹建模、成本测算和绩效评价内容转化为一个可运行、可演示、可用于简历和面试讲解的系统。

项目重点展示以下能力：

1. 对整车物流库区备车作业流程的理解；
2. 对备车道分配与排序问题的建模能力；
3. 对整车短驳运输方案选型的业务分析能力；
4. 对运输成本、效率、安全、质量等指标的量化评价能力；
5. 对 B 端物流调度系统的产品化设计能力；
6. 将算法模型、数据分析与 Web 系统结合落地的能力。

## 4. 业务背景

### 4.1 整车物流园与 VWMS 系统背景

一汽物流智慧物流园是整车自主机厂下线后的第一站，整车会根据客户订单目的地、公路直运、多式联运等不同干线运输方案，匹配到合适的整车物流园区。

整车进入物流园后，仓储管理系统 VWMS 会根据车型、车龄、颜色、目的地等信息，将入库整车自动匹配至园区库位。库位通常带有库位编号、二维码和 RFID 信息标签，倒运司机需要将车辆开至系统指定库位，并通过扫码完成“整车进入库位”操作。

案例 14 中提到，整车库区管理包含三个核心问题：

1. 库位精准匹配问题；
2. 备车道位分配与排序问题；
3. 库区内部倒运路径优化问题。

本项目聚焦其中第 2 个问题：备车道位分配与排序。

### 4.2 备车业务背景

根据 VWMS 系统排布的整车出库时间，需要按照多种约束将待出库整车安排到各备车道，并确定整车在当前备车道内的排序。

主要约束包括：

1. 先进先出原则；
2. 目的地相同或同方向；
3. 车型大小与公路轿运货车装载条件匹配；
4. 公路轿运货车备车顺序；
5. 8 辆整车为一组指定备车道；
6. 整车进入备车道后，必须在 1 小时内完成装载并驶离园区。

因此，备车问题本质上是一个带有业务规则约束的分组、分配与排序问题。其结果会影响后续公路干线运输是否出现多次装卸、车辆等待、装载效率低等问题。

### 4.3 短驳业务背景

随着整车物流从公路直送为主逐步转向铁水干线运输、公路两端短驳的多式联运模式，短驳运输成为整车多式联运中的关键环节。

案例 16 中指出，整车短驳主要用于解决商品车从总装车间到交通枢纽之间的运输问题，也包括目的地交通枢纽到销售网点之间的运输问题。

一汽物流佛山公司原来需要将商品车运至丹灶铁路货运场站，距离整车存储区约 35km。后来佛山市规划并建设官窑货运站及铁路专用线，官窑货运站距离公司整车存储区约 1.4km，并新建整车倒运专用公路。这为整车短驳运输的数智化、无人化和“零公里”改造提供了条件。

短驳运输当前存在两类典型方式：

1. 地跑模式：安排司机将商品车逐一开到铁路货运站，再由车辆将司机接回；
2. 轿运车模式：将多辆商品车装载到轿运车，统一送至铁路货运站。

地跑模式会增加商品车售前行驶里程，也存在一定安全风险；轿运车模式能减少商品车自身行驶里程，但装卸任务由轿运车司机独立完成，作业强度较高，也存在安全生产隐患。

因此，短驳问题需要围绕不同运输距离、道路条件、运输方式、技术方案和成本收益进行综合评价。

## 5. 项目研究范围

本 Web 项目不完整复刻比赛全部内容，而是围绕“备车 + 短驳”两个核心问题进行系统化展示。

### 5.1 案例 14：备车问题

本项目在案例 14 中重点研究：

- 待出库整车如何分组；
- 整车如何分配到备车道；
- 每条备车道内如何排序；
- 如何保证 8 辆整车为一组；
- 如何尽量满足同目的地、同方向、车型装载匹配、先进先出等约束；
- 如何降低后续装车等待、多次装卸和调度混乱。

暂不重点实现：

- 入库库位初次精准分配；
- 巡检后的库位二次调整；
- 库区内部多车路径避碰；
- 无人搬运设备路径规划。

这些内容可以作为后续扩展方向。

### 5.2 案例 16：短驳问题

本项目在案例 16 中重点研究：

- 不同短驳方式的方案选型；
- 地跑、轿运车、无人化/自动化方案的对比；
- 不同距离条件下的适用性分析；
- 短驳运输成本测算；
- 售前行驶里程、安全风险、作业效率和人工投入的评价；
- 建立成本效益和绩效评价指标体系。

## 6. 核心业务问题

### 6.1 备车问题

在给定待出库整车、目的地、车型、颜色、下线时间、出库时间和备车道数量的情况下，系统需要回答：

1. 哪些整车应该被分到同一组？
2. 每组整车应该安排到哪一条备车道？
3. 每条备车道中的整车应该按照什么顺序排列？
4. 是否满足一组 8 辆整车的备车规则？
5. 是否尽量满足同目的地、同方向、车型装载匹配和先进先出原则？
6. 是否能减少后续轿运车装载中的多次调整和等待？

### 6.2 短驳问题

在给定园区到铁路货运站距离、运输需求、车辆能力、人工成本、燃油/电耗成本、装卸成本和安全风险等参数后，系统需要回答：

1. 当前场景更适合地跑、轿运车还是无人化方案？
2. 不同方案的总成本分别是多少？
3. 哪种方案能够最大程度降低商品车售前行驶里程？
4. 哪种方案在效率、安全、成本、可推广性之间综合表现更好？
5. 如果距离从 1.4km、8km 到 35km 变化，方案选择是否会变化？
6. 如何建立可复用的短驳方案成本效益评价模板？

## 7. 系统功能模块

本项目计划实现以下 Web 模块。

### 7.1 首页仪表盘

展示平台整体运行结果，包括：

- 待备车整车数量；
- 已分配备车车辆数量；
- 已使用备车道数量；
- 备车道平均利用率；
- 预计装载完成率；
- 短驳总成本；
- 售前行驶里程减少量；
- 综合绩效得分。

可视化内容包括：

- 备车道利用率柱状图；
- 各目的地车辆数量分布图；
- 短驳方案成本对比图；
- 综合评价雷达图。

### 7.2 数据管理模块

展示和管理模拟数据，包括：

#### 整车数据 car_orders.csv

字段建议：

- car_id：整车编号；
- model_type：车型；
- color：颜色；
- offline_time：下线时间；
- outbound_time：计划出库时间；
- destination：目的地；
- direction：运输方向；
- priority：优先级。

#### 备车道数据 staging_lanes.csv

字段建议：

- lane_id：备车道编号；
- capacity：备车道容量，默认为 8；
- lane_status：备车道状态；
- current_count：当前已分配车辆数；
- assigned_destination：当前目的地或方向。

#### 短驳方案数据 shuttle_schemes.csv

字段建议：

- scheme_id：方案编号；
- scheme_name：方案名称，如地跑、轿运车、无人化轿运车；
- distance_km：短驳距离；
- vehicle_capacity：单次运输车辆数；
- daily_trips：日均趟次；
- vehicle_cost_per_day：车辆日成本；
- driver_cost_per_day：司机日成本；
- loading_cost_per_trip：单趟装卸成本；
- energy_cost_per_km：单位里程能源成本；
- safety_risk_level：安全风险等级；
- automation_level：自动化等级。

### 7.3 备车分配与排序模块

实现一个简化备车算法：

1. 按计划出库时间排序，体现先进先出；
2. 优先将相同目的地或同方向车辆分为一组；
3. 每 8 辆整车形成一个备车组；
4. 将备车组分配到可用备车道；
5. 在备车道内部生成排序结果；
6. 输出每条备车道的车辆列表、目的地、方向、车型结构和装载建议。

第一版可以使用规则/贪心算法实现，后续预留遗传算法、多目标优化或混合整数规划接口。

### 7.4 短驳成本测算模块

支持对不同短驳方案进行成本计算。

成本项可包括：

- 车辆使用成本；
- 司机人工成本；
- 能源成本；
- 装卸作业成本；
- 接驳等待成本；
- 设备折旧或租赁成本；
- 安全风险惩罚成本；
- 售前里程成本。

示例成本函数：

```text
total_cost = vehicle_cost + driver_cost + energy_cost + loading_cost + waiting_cost + risk_cost
```

其中：

```text
energy_cost = distance_km * daily_trips * energy_cost_per_km
```

### 7.5 绩效评价模块

建立多维绩效评价体系，用于比较不同备车方案和短驳方案。

指标建议包括：

#### 成本类指标

- 单车短驳成本；
- 总短驳成本；
- 单日运输成本；
- 人工成本占比；
- 设备成本占比。

#### 效率类指标

- 日均运输能力；
- 单趟运输车辆数；
- 备车道利用率；
- 装载完成率；
- 平均等待时间。

#### 质量类指标

- 售前行驶里程；
- 商品车里程增加量；
- 装卸次数；
- 车辆损伤风险。

#### 安全与智能化指标

- 安全风险等级；
- 人工作业强度；
- 自动化等级；
- 可复制推广性；
- 封闭场景适配性。

#### 综合得分

可以使用简单加权法：

```text
total_score = 0.30 * cost_score
            + 0.25 * efficiency_score
            + 0.20 * quality_score
            + 0.15 * safety_score
            + 0.10 * automation_score
```

第一版中可以使用人工设定权重，后续可扩展为 AHP、熵权法、TOPSIS 或多目标评价模型。

### 7.6 方案对比模块

对比以下方案：

#### 方案 A：传统地跑模式

特点：

- 商品车逐一被司机开到铁路货运站；
- 售前行驶里程增加；
- 人工投入较高；
- 组织方式简单，但安全风险和人力成本较高。

#### 方案 B：传统轿运车模式

特点：

- 一台轿运车可装载多辆商品车；
- 可降低商品车自身行驶里程；
- 装卸作业对司机要求较高；
- 适合中短距离和一定规模的倒运任务。

#### 方案 C：无人化/自动化短驳方案

特点：

- 适合封闭或半封闭道路环境；
- 可降低人工投入；
- 前期设备投入较高；
- 适合标准化程度高、路线稳定、距离较短的场景。

对比指标包括：

- 总成本；
- 单车成本；
- 日运输能力；
- 售前行驶里程；
- 安全风险；
- 人工投入；
- 自动化等级；
- 综合绩效得分。

## 8. 第一版 MVP 实现边界

第一版 MVP 不追求完整工业级优化，只需要实现清晰、可运行、可解释的简化模型。

### 8.1 第一版必须实现

1. 读取模拟整车、备车道、短驳方案数据；
2. 展示数据管理页面；
3. 根据出库时间、目的地、方向进行备车分组；
4. 将每组车辆分配到备车道；
5. 输出备车排序结果；
6. 对地跑、轿运车、无人化方案进行短驳成本测算；
7. 建立多维绩效评价指标；
8. 展示优化前后或不同方案对比；
9. 提供中文 README 和中文注释。

### 8.2 第一版暂不实现

1. 真实 VWMS 系统对接；
2. 真实地图路径规划；
3. 多车辆路径避碰；
4. 无人驾驶仿真；
5. 真实 TruckSim 模型；
6. 复杂登录权限；
7. 真实企业数据导入；
8. 数据库持久化；
9. 工业级精确优化求解。

## 12. 面试讲解口径

本项目可以这样讲：

> 这个项目来源于全国大学生物流设计大赛的一汽物流案例。原案例文件包含多个题目，我们团队选择其中与智慧物流园备车管理、整车短驳运输零公里改造、成本效益与绩效评价相关的题目构成综合方案。
>
> 在案例 14 中，我重点关注备车问题。根据 VWMS 系统的出库计划，需要将待出库整车按照先进先出、目的地相同或同方向、车型装载匹配、公路轿运货车装车顺序等约束分配到备车道，并确定每条备车道内的排序。由于一组通常为 8 辆整车，且进入备车道后需要在 1 小时内完成装载并驶离园区，因此该问题可以抽象为带业务规则的分组、分配与排序优化问题。
>
> 在案例 16 中，我重点关注短驳问题。整车物流正在从公路直送转向多式联运，短驳运输成为影响成本、效率、安全和售前里程的重要环节。佛山场景中，原先到丹灶铁路货运站距离约 35km，后来新建官窑货运站后距离缩短至约 1.4km，这为轿运车、无人化和自动化短驳方案提供了应用条件。因此我设计了短驳方案选型、成本测算和多维绩效评价模块。
>
> 为了将比赛方案转化为可展示成果，我进一步设计了一个 B/S 架构 Web Demo，包含首页仪表盘、数据管理、备车分配、短驳成本测算、绩效评价和方案对比模块。第一版使用模拟数据和贪心/规则算法实现，后续可以扩展遗传算法、多目标优化和地图可视化。

## 13. 后续扩展方向

1. 引入遗传算法，对备车道分配和排序进行全局优化；
2. 建立混合整数规划模型，约束包括备车道容量、出库时间、目的地、装车顺序等；
3. 引入多目标优化，同时考虑成本、效率、等待时间和装载便利性；
4. 增加地图可视化，展示园区、备车道、货运站和短驳线路；
5. 引入 TOPSIS、AHP 或熵权法完善绩效评价；
6. 将 CSV 数据升级为 SQLite 或 MySQL；
